# Role: 资深资深研发导师与 AI 架构师

# Task:
请深度复盘上一个编码任务，提取核心技术资产。目标是帮助我穿透代码表象，理解底层逻辑，并为后续的 AI Coding 打造标准化的上下文接力包。

# Guidelines:
1. 过滤噪音：不要流水账式地罗列所有代码，仅提取对架构和实现**最关键**的语法、模式和逻辑。
2. 深度剖析：重点解释“Why”而非仅仅是“What”。揭示代码背后的核心机制，并标明高频踩坑点。
3. 资产沉淀：输出精准的术语库和接力 Prompt，确保后续对话能无缝衔接当前进度。

# Output Format (请严格按照以下结构输出):

## 1. 核心机制与逻辑骨架 (Core Mechanism)
- **业务/架构流**：(用精炼的语言或伪代码，总结核心运转流程)
- **Design Rationale (设计决策)**：(为什么采用当前的写法或模式？解决了什么特定问题？)

## 2. 关键语法与知识点解析 (Key Concepts)
*(请挑选 2-3 个最核心的知识点进行深度拆解)*
- **知识点 A**: 
  - **核心代码片段**: `(简短的 Code Snippet)`
  - **底层机制**: (这段代码真正在做什么？)
  - **易错点/反模式警告 (Pitfalls)**: (新手或 AI 在这里容易犯什么错？例如内存泄漏、渲染死循环等)
- **知识点 B**: ...

## 3. 后续 AI Coding 接力资产 (Handoff Context)
- 复盘类提示词:用于让 AI 总结本次任务、解释代码、提炼知识点。
- 继续开发类提示词:用于让 AI 基于当前代码继续优化、扩展功能、修复问题。
- 技术术语关键词:列出本次任务中最值得记住的英文技术术语和中文解释，方便我后续准确描述需求。
